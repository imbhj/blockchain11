## middleware?

=> 요청과 응답
=> 응답을 하기전, 검증이 필요하다!
=> 요청과 응답에 대한 흐름을 가로채서 검증을 한다.

## 활용 기준?

- 이 요청을 보낸 사용자가 로그인했는가?
- 요청한 데이터가 올바른 형식인가?
- 이 요청을 로그로 기록해야 하는가?

=> 역할의 분리 => 기능은 기능대로, 검증은 검증대로
